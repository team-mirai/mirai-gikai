# 沼津市議会データソース調査と反映方針

## 1. 目的

沼津市版みらい議会を構築する前提として、沼津市議会の公開情報のうち
どこを一次データソースにするかを整理し、現行 `mirai-gikai` に
どう反映するかを検討する。

調査日は 2026年3月17日。

## 2. 結論

沼津市版の中核データは、次の 3 系統を一次ソースにするのが最も現実的。

1. 沼津市議会公式サイトの静的 HTML
2. そのページからリンクされる議案 PDF / 日程 PDF
3. 会議記録検索システムと中継ページへの外部リンク

特に重要なのは `本会議のお知らせ` で、ここに会期日程、提出議案 PDF、
一般質問 PDF、議案質疑 PDF、議事報告がまとまっている。
沼津版の MVP は、このページを起点に議案一覧と会期情報を組み立てるのがよい。

今回確認した範囲では、議案単位の CSV / JSON API は見当たらなかった。
したがって、初期段階は `HTML + PDF 抽出` を前提に設計すべき。

## 3. 調査した主な公式ソース

### 3.1 沼津市議会トップ

- URL: `https://www.city.numazu.shizuoka.jp/shisei/g-shigiki/g-sigiki/index.htm`
- 役割:
  - 市議会関連ページのハブ
  - 新着情報
  - 議員一覧
  - 会派・委員会別名簿
  - 本会議のお知らせ
  - 本会議の報告
  - 委員会・全員協議会等の日程
  - 会議録検索
  - 議会中継
- 形式: 静的 HTML
- 評価:
  - URL 構造が比較的素直で取得しやすい
  - 市議会関連情報のハブとして最重要

### 3.2 定例会会期予定

- URL: `https://www.city.numazu.shizuoka.jp/shisei/g-shigiki/g-sigiki/annai/yotei.htm`
- 2026年3月17日時点の更新日: 2026年1月30日
- 取得できる情報:
  - 定例会ごとの開会予定日 / 閉会予定日
  - 年間予定表 PDF
- 形式: HTML + PDF
- 使い道:
  - `diet_sessions` 相当の会期マスタ作成
  - トップページの「今後の定例会」表示
- 注意点:
  - 予定ベースであり、確定日程ではない

### 3.3 本会議のお知らせ

- URL: `https://www.city.numazu.shizuoka.jp/shisei/g-shigiki/g-sigiki/annai/oshirase.htm`
- 2026年3月17日時点の更新日: 2026年3月11日
- 取得できる情報:
  - 当該定例会の会期日程 PDF
  - 提出議案 PDF 一覧
  - 一般質問 PDF
  - 議案質疑 PDF
  - 各日の議事報告テキスト
  - 追加議案の注記
- 形式: HTML + PDF
- 評価:
  - 沼津版みらい議会の最重要ソース
  - 議案番号、正式議案名、PDF URL、追加議案日が取れる
  - 議事報告から「委員会付託」「質疑実施」などの進捗が読める

#### 実際に確認できた構造

- `nittei-0802.pdf` のような会期日程 PDF がある
- `gian0802/gi-4.pdf` のような議案 PDF が個別リンクされる
- リンクテキストに `議第4号 ...` のような議案番号・正式名称が含まれる
- `【令和8年2月9日に追加された議案】` のような追加注記が HTML に含まれる

#### PDF の機械抽出可否

サンプル PDF を `pdftotext` で確認したところ、少なくとも以下は抽出可能だった。

- 議案番号
- 正式題名
- 提出日
- 提案者
- 条例改正文や提案理由

そのため、初期実装から PDF をテキスト抽出対象に入れてよい。

  - 過去会期アーカイブ

- 注意点:
  - 最新会期の詳細は `本会議のお知らせ` に先に出ることがある

### 3.5 委員会・全員協議会等の日程

- URL: `https://www.city.numazu.shizuoka.jp/shisei/g-shigiki/g-sigiki/annai/nittei.htm`
- 2026年3月17日時点の更新日: 2026年3月9日
- 取得できる情報:
  - 年度単位の委員会等日程 PDF
- 形式: HTML + PDF
- 使い道:
  - 委員会日程カレンダー
  - 議案詳細に「今後の審査予定」を紐づけるための基礎データ
- 注意点:
  - 勉強会や協議案件等は原則掲載しないと明記されている

### 3.6 会議記録検索システム

- URL: `https://ami-search.amivoice.com/numazu/usr/search.exe`
- 形式: CGI ベースの HTML
- 取得できる情報:
  - 会期一覧
  - 会議記録一覧
  - 簡易検索 / 詳細検索
  - 委員会会議録
- 今回確認できたこと:
  - 会期一覧ページに `vcsm=...` が出る
  - 会議記録一覧ページに `vcsv=...` が出る
  - 委員会会議録も検索対象になっている
- 使い道:
  - 議案ページから会議録への外部リンクを張る
  - 将来的な議案別発言抽出の基盤

#### 初期段階での扱い

初期リリースでは全文取込までは行わず、まずは

- 会議録検索ページへの導線
- 会期別会議一覧への導線

に留めるのが妥当。

理由:

- CGI パラメータ依存で取得実装の保守コストがやや高い
- 発言単位の構造化には追加の解析が必要
- まずは議案 PDF と会期情報だけで公開価値が高い

## 4. 法務・運用上の注意

### 4.1 著作権

沼津市の `著作権について` では、ホームページ上のテキスト、画像、PDF などの
著作権は沼津市に帰属し、私的使用や引用の範囲を超える転載・二次利用には
事前許可が必要とされている。

したがって、沼津版みらい議会では以下を原則にする。

- 原典 PDF そのものの再配布は行わない
- 原典 URL へのリンクを表示する
- 本文は要約・構造化メタデータ・引用最小限で扱う
- フルテキスト公開は別途利用条件確認が必要

### 4.2 個人情報

議員一覧ページの住所・電話番号は取り込まず、
氏名、会派、委員会、顔写真程度に限定する。

### 4.3 ソースの永続性

- 市公式 HTML/PDF: 比較的安定
- AmiVoice 会議録検索: URL パラメータが特殊
- DiscussVision / YouTube: 外部サービス依存が強い

そのため、システム内の正本は

- 会期メタデータ
- 議案メタデータ
- 抽出済み要約
- 原典 URL

に置き、動画や会議録検索は参照リンク扱いに留めるべき。

## 5. 現行 mirai-gikai へのマッピング

### 5.1 そのまま活かせるもの

- `bills`: 議案 1 件 = `bills` 1 件で流用可能
- `bill_contents`: 市民向けタイトル、要約、解説本文の格納先として有効
- `tags`: 政策分類に流用可能
- `diet_sessions`: 内部名は維持しつつ「定例会/臨時会」に再定義可能

### 5.2 そのままでは合わないもの

#### `originating_house`

自治体議会には衆議院/参議院の概念がないため不適合。

#### `shugiin_url`

名称も意味も合っていない。
`source_url` への改名または原典複数管理テーブルへの移行が必要。

#### `bill_status_enum`

国会向けステータスは地方議会向けには粗すぎる。
地方議会向けの進行状態への再設計が必要。

## 6. 推奨取り込みフロー

### Phase 1: 手動 + 半自動のMVP

1. `本会議のお知らせ` から当該会期を取得
2. 提出議案リンク一覧を抽出
3. 各 PDF を保存せず URL と抽出テキストだけを保持
4. 管理画面で議案レコードを作成
5. `bill_contents` に市民向け要約を登録
6. 原典 PDF / 会議録 / 動画を外部リンクとして表示

### Phase 2: 定期取り込み

1. 会期中は `oshirase.htm` を定期チェック
2. PDF リンク一覧の差分を検出
3. 追加議案や更新日変更を検知
4. `status` と `status_note` を自動更新

### Phase 3: 会議録連携

1. AmiVoice 側の会期一覧・会議一覧を巡回
2. 会議録公開を検知
3. 議案番号やキーワードで該当議案に紐づけ
4. 議案ページに関連会議録や発言を出す

## 7. 最終提案

沼津版みらい議会では、初期実装を次の形にするのがよい。

1. `annai/oshirase.htm` をコアデータ源にする
2. `annai/yotei.htm` と `annai/nittei.htm` で会期・委員会情報を補う
3. 議案 PDF を抽出して `bill_contents` の下書き生成に使う
4. 会議録検索・動画は外部リンクとして統合する
5. 原典は再配布せず、要約 + 原典リンク方式を徹底する
6. 将来的に `bill_source_documents` と `council_meetings` を追加する

この進め方なら、現行 `mirai-gikai` の構造を活かしながら、
最小コストで「沼津市議会の議案を追えるサービス」に寄せられる。
